# Деплой тестовой страницы на прод и проверка тегов

## Цель

Задеплоить тестовую страницу конструктора (`/8march_design/`) на прод и проверить, что блоки включаются и выключаются через админку (теги в контейнерах «Новый дизайн»).

## 1. Деплой кода

- Смержить ветку `tags` в ветку, которую деплоите на прод (у вас уже идёт merge).
- Запушить и задеплоить на прод как обычно (capistrano / скрипт / панель).

## 2. На сервере после деплоя

### 2.1 Миграции

Чтобы на проде появились контейнеры «Новый дизайн: блоки главной страницы» и «Новый дизайн: блоки футера»:

```bash
python manage.py migrate
```

Миграция `0066_ensure_new_design_middle_container` создаст эти контейнеры, если их ещё нет.

### 2.2 Показать контейнеры в админке

В админке в разделе «Контейнеры шаблонных тегов» отображаются только те контейнеры, которые перечислены в настройке **Constance** → **DISPLAY_TAG_CONTAINERS**.

Чтобы появились пункты для управления блоками тестовой страницы, в **DISPLAY_TAG_CONTAINERS** должны быть строки:

- `new_design_middle`
- `new_design_bottom`

**Вариант А — через админку:**  
Зайти в **Constance** (или «Конфигурация»), найти **DISPLAY_TAG_CONTAINERS** и задать значение, например:

```json
["middle", "bottom", "new_design_middle", "new_design_bottom"]
```

(Точный формат зависит от того, как у вас настроен Constance — JSON или список через запятую.)

**Вариант Б — с сервера одной командой:**

```bash
python manage.py set_display_tag_containers
```

Команда выставит: `["middle", "bottom", "new_design_middle", "new_design_bottom"]`.

После этого в админке появятся:

- **Новый дизайн: блоки главной страницы** (`new_design_middle`)
- **Новый дизайн: блоки футера** (`new_design_bottom`)

## 3. Открыть тестовую страницу на проде

URL тестовой страницы:

```
https://ВАШ-ДОМЕН/8march_design/
```

На проде маршрут тот же, что и локально; отдельно включать его не нужно. Ссылку на эту страницу с главной можно не давать — заходить по прямой ссылке для проверки.

## 4. Проверка: включение и выключение тегов

1. Зайти в админку → **Контейнеры шаблонных тегов**.
2. Открыть **«Новый дизайн: блоки главной страницы»**.
3. В блоке «Шаблонные теги в контейнере»:
   - **Добавить** нужные теги (герой, акции, рубрики, коллекция, отзывы, карта и т.д.) из выпадающего списка «Шаблонный тег».
   - **Порядок** меняется кнопками **↑** и **↓** (и полем «Позиция»).
   - Чтобы **выключить** блок — удалить строку с этим тегом или не добавлять его.
4. Сохранить контейнер.
5. Открыть (или обновить) **https://ВАШ-ДОМЕН/8march_design/** — состав блоков должен соответствовать настройкам: добавленные теги отображаются, удалённые — нет.

Аналогично проверяется контейнер **«Новый дизайн: блоки футера»** — блоки под основной частью страницы.

## 5. Если контейнеры пустые

После первой миграции контейнеры создаются **пустыми**. На проде нужно один раз вручную добавить в них те блоки и порядок, которые нужны (как делали локально). Данные из локальной БД на прод не переносятся автоматически.
